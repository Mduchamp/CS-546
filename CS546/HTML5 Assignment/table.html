This is a table. It has several columns. Some cells can take up more than one column.
It has rows too! I can have as many of either as it wants. The cells are just text boxes with certain properties. The table declaration takes care of the exact size and location.
Cells can also take up multiple rows. This is harder to deal with. Because tables are declared row-by-row, except where a cell takes multiple rows. Then, a cell on the next row is automatically declared.
See? This is the second cell! This also happens with columns, but they are in the same row. And that's the basics of tables.
